자산건전성 분류 자동화 PM

프로젝트 개요

  • 자산건전성 분류 업무를 자동화하여 수작업 의존도를 줄이고 분류 정확성과 처리 효율성을 향상시키기 위한 프로젝트

기간

2024.07~2025.02

담당 역할

  • 업무 전과정 수행

    • *요구사항 분석 → 업무 협의 → DB 설계 → 개발 → 테스트 → 운영

기술 및 개발 환경

  • Language : Java, PL/SQL
  • Framework : Spring
  • Database : Oracle
  • Middleware : JEUS, TeraStream
  • Solution : BXM, Frism, WebSquare, G-Form

시스템 구조

사용자 → UI서버(WebSquare) → APP서버(Service) → 배치서버(TeraStream) → 결과 파일 출력

주요 기능

  • 환원데이터 배치 적재
  • 기초데이터 업로드
  • 자산건전성 자동분류
  • 기초데이터 업로드 상태 조회
  • 자산건전성 분류 결과 수정

기초데이터 업로드 및 상태 조회 기능 제공

기술적 이슈 및 해결

문제1

  • 상황

    • 업로드 건수가 6만건 이상이 되면서 10분 내외 배치 속도 발생

  • 원인

    • 건별 insert

  • 해결

    • - 건별 insert에서 bulk insert로 변경

    - commit interval 값 최적화

  • 결과

    • 배치수행 속도 개선(3분24초 → 1분34초)


문제2

  • 상황

    • 업로드 파일이 파일최대 사이즈인 50MB 초과하여 업로드 불가

  • 원인

    • 파일 최대 사이즈

  • 해결

    • - http.m Limit RequestBody 값 수정(50MB → 100MB)

    - WebToB서버 설정파일 변경 후 재기동 수행

  • 결과

    • 업로드 오류 개선

성과

  • 자산건전성 데이터의 체계적 저장 및 관리 구현
  • 현업의 수작업 처리시간 단축(3영업일 → 1분)
  • 휴먼 오류 감소

회고

배운점

  • Oracle PL/SQL을 통한 데이터 처리 경험
  • TeraStream을 활용한 ETL변환 경험
  • Spring 기반 SC Service 구조에서의 요청/응답 처리 이해
  • WebSquare UI와 APP서버 간 연계 구조 이해
  • 복잡한 결과에 있어 분류 원인의 추적 관리가 중요성을 이해

아쉬운점

  • 단위테스트 부족으로 인한 운영 에러 발생
  • 에러 확인 서비스 로직 추가 필요
  • 현업과 요건 간 우선순위 적용 방식에 대한 기준을 사전에 충분히 정리하지 않아 개발단계에서 추가 협의 발생

채권 문자 자동발송 개발

프로젝트 개요

  • 채권관리팀에서 매일 발송하는 필수 및 독촉 안내 문자 발송 업무를 자동화하여 수작업 의존도를 줄이고 발송 정확성과 업무 효율성을 향상시키기 위한 프로젝트

기간

2025.03~지속

담당 역할

  • 개발 업무 수행

    • 요건 분석 → 쿼리 작성 → 개발 → 테스트 → 운영

기술 및 개발 환경

  • Language : Java, SQL
  • Framework : Spring
  • Database : Oracle
  • Middleware : JEUS
  • Solution : BXM, Frism, WebSquare, G-Form, JOBPASS, SMSagent

시스템 구조

  • 수기 발송

    • 사용자 → UI서버(WebSquare) → APP서버(Service) → 배치서버(sh) → DB insert

  • 자동 발송

    • 잡패스서버(Jobpass) → 배치서버(sh) → DB insert

주요 기능

  • 문자별 대상자 조회
  • 문자 테스트 발송
  • 문자 수기 일괄 발송
  • 문자 자동 배치 발송
  • 처리 결과 조회

주요 화면

발송 화면

문자별 발송 대상자 조회 및 일괄 발송 기능, 처리상태 조회 기능 제공

기술적 이슈 및 해결

문제1

  • 상황

    • Batch reader에서 평일이면서 영업일이 아닌 경우 배치 실패

  • 원인

    • 비영업일이면 retrun값 없이 reader 종료

  • 해결

    • - 평일이면서 영업일이 아닌 경우 배치 실패 처리되지 않도록 return값을 null로 추가하여 reader stepd을 Complite시킴

  • 결과

    • retrun null로 인해 후속 procession 수행되지 않고 정상 종료 처리


문제2

  • 상황

    • 실시간 중앙회 전문통신 시 여신 원장 업데이트 항목에 차기원금납입일이 누락되어 연체시작일이 오조회 됨

  • 원인

    • 환원데이터와 실시간데이터의 D-1 간극

  • 해결

    • - 전문송신 받은 차기원금납입일 항목을 원장에 업데이트하여 납입예정일을 정상 산출함

    - WebToB서버 설정파일 변경 후 재기동 수행

  • 결과

    • 연체시작일 정상 조회

성과

  • 현업의 수작업 처리시간 단축(1시간 → 1분)
  • 휴먼 오류 감소

회고

배운점

  • 외부 발송 시스템 전송 과정 경험을 통한 시스템 인터페이스 구조의 이해
  • 여신 후단 업무 요건을 분석하는 과정에서 채권관리 업무 프로세스에 대한 이해도 상승

아쉬운점

  • 발송결과 모니터링 및 오류 확인을 위한 화면 추가 필요
  • 테스트 단계에서 예외 케이스를 체계적으로 정리하지 못해 테스트 과정에서 추가 보완 발생
